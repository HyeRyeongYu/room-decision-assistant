# 1. Business purpose

- 예전부터 오늘날까지 2030세대는 학업과 취업 등의 이유로 자취를 시작하는 경우가 많다. 이 과정에서 원룸, 오피스텔 등의 다양한 주거 공간 중 하나를 선택해야 하며, 주거 환경을 직접 확인하고, 계약을 진행하는 과정에서 여러 요소에 관한 판단과 의사결정 상황에 놓이게 된다. 그러나, 관련 경험과 정보의 부족 그리고 전문 용어에 대한 이해 부족으로 인해 계약 과정에서 어려움을 겪는 경우가 많다. 그 결과, 계약을 체결한 세입자(임차인)는 이전에 충분히 파악하지 못했던 문제를 뒤늦게 발견하고, 이를 해결해야 하는 상황에 직면하기도 한다.
- 특히, 현행 부동산 거래 과정에서는 공인중개사가 계약 체결 직전에 ‘중개대상물 확인·설명서’를 제공할 의무가 있다. (공인중개사법 제25조) 하지만, 소비자가 여러 매물을 탐색하고, 비교하는 초기 단계에서는 공인중개사가 일부 항목을 구두로 설명하는 경우는 있으나, 주거 환경을 체계적으로 기록하고 비교할 수 있는 서면의 형태는 제공하지 않는다. 따라서, 소비자는 인터넷 검색이나 개인적인 경험에 의존하여 체크리스트를 준비하기에, 확인해야 하는 중요 항목을 놓친 채 의사결정을 내리는 문제가 발생하기도 한다. 
- 또한, 기존에 블로그나 유튜브 등의 매체에서 찾을 수 있는 체크리스트는 대부분 여러 매물을 비교할 수 있는 형태가 아닌, 단일 매물을 기준으로 체크리스트가 구성되어 있어, 종합적으로 매물들의 장단점을 비교하고, 판단하는 과정에 한계가 있다. 그리고 최종적으로 결정하기 전까지 여러 매물의 정보를 소비자가 개별적으로 관리하고 보관해야 하는 불편함이 존재한다.
- 따라서, 본 프로젝트에서는 자취방 선택 과정에서 발생하는 문제들을 해결하기 위해, 사용자가 다양한 매물을 체계적으로 평가하고, 비교할 수 있도록 웹 기반의 의사결정 지원 시스템을 개발하고자 한다. 이 시스템은 중개대상물 확인·설명서의 항목과 내용을 바탕으로 체크리스트 기능을 제공하며, 각 매물에 대한 사용자의 평가 결과를 점수로 정량화하여 의사결정의 정확도를 높이고, 중요한 항목의 확인 누락을 방지함으로써, 중개 계약 이후의 리스크를 줄이는 것을 목표로 한다.

# 2. System context diagram

![System Context Diagram](./system-context-diagram.png)

Figure 1. System Context Diagram

### (1) User → System
- Input checklist items : 방(매물) 평가를 위한 체크리스트 항목 입력
- Manage and Select rooms : 여러 방 후보 선택 및 관리
- Record observations : 각 방에 대한 관찰 사항 및 메모 기록
- Assign scores : 체크리스트 기준에 따른 평가 점수 부여
- Request comparison : 서로 다른 방 후보 간의 비교 요청

### (2) System → User
- Provide checklist : 법적 가이드라인(중개대상물 확인·설명서)에 기반한 구조화된 체크리스트 제공
- Calculate evaluation scores : 각 방에 대한 항목 평가 점수 계산 제공
- Compare room candidates : 다수의 방 후보 간 비교 결과 제공
- Suggest choice : 사용자 입력 및 평가를 기반으로 최적의 방 추천
- Calculate estimated cost : 주거 관련 예상 비용 계산
- Provide customizable checklist : 사용자 맞춤 체크리스트 항목 제공

## 4. Concept of operation
### (1) Add room candidate
| Category | Description |
|:----------:|-------------|
| Purpose | 사용자가 비교를 위해 새로운 방의 체크리스트를 시스템에 등록하기 위함 |
| Approach | 프로그램의 홈 화면에서 체크리스트 추가 기능을 선택하면, 추가하고자 하는 매물의 유형을 선택할 수 있다. 이후 선택된 유형에 맞는 체크리스트 형식이 제공되며, 해당 매물에 대한 기본 정보를 입력할 수 있다. 생성 기능을 선택하면 입력된 정보가 저장되고 새로운 방 후보로 등록된다. |
| Dynamics | 사용자가 방의 체크리스트를 추가하고 싶은 경우 |
| Goals | 사용자가 새로운 방 후보에 대한 체크리스트를 생성할 수 있도록 한다. |

### (2) Evaluate room with checklist  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 체크리스트 항목을 기준으로 방을 체계적으로 평가하기 위함 |
| Approach | 체크리스트가 제공되면 각 항목에 대해 상/중/하와 같은 평가 기준 중 하나를 선택하여 평가를 수행할 수 있다. 이후, 총점 계산 기능을 선택하면, 평가 결과가 반영되어 총점수가 계산된다. 만약 필수 항목의 평가가 누락된 경우, 누락된 항목의 그룹 번호와 항목 번호가 안내된다. |
| Dynamics | 사용자가 방의 체크리스트를 평가하고자 하는 경우 |
| Goals | 사용자가 매물 계약 체결 전 필수 항목을 누락 없이 확인하고 평가할 수 있도록 한다. |

### (3) Customize checklist 
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 필수 항목 외에 원하는 항목을 체크리스트에 추가하기 위함 |
| Approach | 기본 체크리스트에 추가로 맞춤 항목 입력을 원하는 경우, 체크리스트 항목 추가 기능을 선택할 수 있다. 이후 그룹 설정과 항목 내용을 입력할 수 있는 화면이 제공되며, 원하는 항목을 입력하여 추가할 수 있다. 입력된 항목은 체크리스트에 반영되고, 추가된 항목에 대해서도 평가를 수행할 수 있다. |
| Dynamics | 사용자가 체크리스트에 항목을 추가 및 평가하고자 하는 경우 |
| Goals | 사용자가 개인의 기준에 맞는 항목을 추가하여 체크리스트를 확장할 수 있도록 한다. |

### (4) Enter housing cost information  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 주거 비용을 체크리스트에 입력하기 위함 |
| Approach | 계약 유형에 따라 생성된 체크리스트에서 주거 비용 항목을 선택하면 비용을 입력할 수 있는 입력란이 제공된다. 필수 항목인 주거 비용을 입력하면 해당 정보가 시스템에 반영된다. 이후 입력된 비용 정보가 저장되고, 매물 비교 시 정량적으로 활용된다. |
| Dynamics | 사용자가 방의 주거 비용 정보를 입력하고자 하는 경우 |
| Goals | 사용자가 주거 비용을 입력하여 매물 간의 비용을 정량적으로 비교할 수 있도록 한다. |

### (5) Compare rooms  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 여러 후보 방을 체크리스트 평가 결과를 기반으로 비교하기 위함 |
| Approach | 매물 비교 기능을 선택하면, 각 매물의 체크리스트 평가 결과를 기반으로 시스템이 비교 결과를 제공한다. 이후 총점 기반 비교와 상위 평가 항목 개수 기반 비교 중 하나를 선택할 수 있으며, 선택된 기준에 따라 결과가 제시된다. 이러한 비교 결과를 통해 매물 간의 차이를 확인할 수 있다. |
| Dynamics | 사용자가 매물 간의 비교를 하고 싶은 경우 |
| Goals | 사용자가 다양한 기준으로 매물을 비교하여 최적의 선택을 할 수 있도록 한다. |

### (6) View recommended room  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 시스템이 추천한 매물 정보를 확인하기 위함 |
| Approach | 추천 기능을 선택하면, 각 매물의 평가 총점을 기반으로 비교가 수행된다. 이후 사용자 맞춤 항목이 존재하는 경우 해당 항목이 추가적으로 고려되며, 유사한 평가 결과를 가진 매물에 대해서는 주거 비용이 반영된다. 이러한 과정을 통해 가장 적합한 매물이 추천된다. |
| Dynamics | 사용자가 추천 후보 매물을 알고 싶은 경우 |
| Goals | 사용자가 평가 결과를 기반으로 맞춤형 매물을 추천받을 수 있도록 한다. |

### (7) Record room observations  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 후보 매물에 대한 관찰 사항 등을 기록하기 위함 |
| Approach | 체크리스트의 메모 항목을 선택하면, 매물 확인 후의 생각이나 특이 사항을 입력할 수 있는 입력란이 제공된다. 이후 해당 내용을 자유롭게 작성하면, 입력된 정보가 체크리스트에 반영된다. |
| Dynamics | 사용자가 매물의 추가적인 내용을 기록하고 싶은 경우 |
| Goals | 사용자가 항목 평가 외의 내용을 기록하여 보다 종합적인 매물 평가를 할 수 있도록 한다. |

### (8) Save evaluation results  
| Category | Description |
|----------|-------------|
| Purpose | 사용자가 체크리스트 평가 결과와 기록 내용을 저장하기 위함 |
| Approach | 체크리스트 저장 기능을 선택하면, 현재까지 입력된 필수 항목 평가, 사용자 추가 항목 평가, 주거 비용 정보 및 메모 내용이 하나의 체크리스트 기록으로 반영된다. 이후 저장된 기록은 시스템에 보관되어 사용자가 다시 확인하거나 재평가에 활용할 수 있다. |
| Dynamics | 사용자가 평가 결과를 저장하고 이후에 다시 확인하고 싶은 경우 |
| Goals | 사용자가 체크리스트 평가 결과를 저장하여 추후 비교 및 재평가에 활용할 수 있도록 한다. |
